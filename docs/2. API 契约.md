# Satori API 契约

本文档定义当前版本前后端联调使用的 HTTP API 契约。

当前版本面向 Gemini 负责的前端集成，后端提供稳定的路径、查询参数和 JSON 响应结构。

## 1. 基本约定

- Base URL：`http://127.0.0.1:3000`
- 协议：HTTP
- 数据格式：`application/json`
- 字符编码：UTF-8

当前版本只提供只读接口，不涉及鉴权。

## 2. 健康检查

### `GET /api/health`

用于检查服务是否可用。

成功响应：

```json
{
  "status": "ok"
}
```

机器可读示例文件：

- `docs/api-examples/health.json`

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `status` | `string` | 当前固定为 `ok` |

## 3. 搜索接口

### `GET /api/search`

根据白话、黑话或场景描述搜索词条。

查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `q` | `string` | 是 | 搜索关键词。去除首尾空白后不能为空，最大 200 个字符 |
| `limit` | `integer` | 否 | 返回结果数量。默认 `10`，超过 `50` 会被截断到 `50`，小于 `1` 会被提升到 `1` |

成功响应示例：

```json
{
  "query": "大家先统一想法",
  "results": [
    {
      "id": "jargon_lar_tong_dui_qi",
      "term": "拉通对齐",
      "plain": "大家先统一想法",
      "explanation": "让相关的人先把目标、分工和时间说清楚。",
      "examples": ["这个需求先拉通对齐一下。"],
      "tags": ["职场", "会议", "协作"],
      "score": 1.0
    }
  ]
}
```

机器可读示例文件：

- `docs/api-examples/search-success.json`

响应字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `query` | `string` | 归一化后的查询文本 |
| `results` | `SearchResult[]` | 按相关性降序排列的结果列表 |

`SearchResult` 字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `string` | 词条唯一标识 |
| `term` | `string` | 黑话或梗本身 |
| `plain` | `string` | 对应的人话表达 |
| `explanation` | `string` | 解释说明 |
| `examples` | `string[]` | 例句列表 |
| `tags` | `string[]` | 标签列表 |
| `score` | `number` | 当前排序分数，范围为 `0.0` 到 `1.0` |

无匹配时返回空数组：

```json
{
  "query": "完全不存在的说法",
  "results": []
}
```

## 4. 错误响应

所有当前由后端显式处理的请求错误都返回 JSON：

```json
{
  "error": "invalid_query",
  "message": "q must be present, non-empty, and within the character limit"
}
```

机器可读示例文件：

- `docs/api-examples/error-invalid-query.json`

错误码说明：

| HTTP 状态码 | `error` | 说明 |
| --- | --- | --- |
| `400` | `invalid_query` | `q` 缺失、为空白，或超过最大字符数 |
| `400` | `invalid_limit` | `limit` 不是整数 |

`invalid_limit` 响应示例：

```json
{
  "error": "invalid_limit",
  "message": "limit must be an integer between 1 and 50"
}
```

机器可读示例文件：

- `docs/api-examples/error-invalid-limit.json`

## 5. 当前实现说明

- 当前搜索实现基于本地 JSON 语料和简单关键词排序。
- 当前结果按 `score` 降序排序；同分时按 `term` 升序排序。
- `score` 目前用于排序和展示，相似度语义后续可能调整，但字段会保留。
- 后续切换到 Embedding 和 LanceDB 时，路径和基础响应结构应保持兼容；如有破坏性变更，需要先更新本文档。
